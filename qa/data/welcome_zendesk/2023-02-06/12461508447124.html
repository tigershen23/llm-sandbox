Hubspot Integration Documentation

Overview

Welcome’s Hubspot integration enables our customers to use the actions a Contact takes in Welcome to inform the next step of that Contact’s buyer’s journey.

The integration is streamlined to require minimal configuration. It makes it so your Hubspot Contacts need only click a link to get into an event, without going through registration. It sends back clear, easy-to-interpret signals of attendee sentiment, in real-time, for follow-up action. Some examples of things you can do with the Welcome integration are:

  • Signal an Account Executive in real time when their prospect is attending one of your Webinars, or when a qualified account engages with a “Do you want to talk to a Salesperson?” Poll 

  • Increase the Hubspot lead score by 25 points for any lead which has spent more than 10 minutes at a specific Webinar.

  • Create new Contacts for people who attend your webinars with their custom registration fields 

 

Note to Previous Hubspot Integration Users

If you were using the previous version of our Hubspot integration which leverages the built-in Welcome Properties – Invited Welcome Events, Registered Welcome Events, and Attended Welcome Events – the new integration does NOT update these fields anymore. 

Setup – Connecting Welcome to your Hubspot Instance 

  1. Log in to your Welcome Producer Account at app.experiencewelcome.com
  2. Navigate to Integrations
  3. Click configure
  4. Log in to your Hubspot Instance in the window that pops up. 
    1. Enter any name for your authentication and Click Create. Leave the Developer API Key field blank.
    2. Choose the Hubspot account you want to connect to.
    3. Confirm with Hubspot that Welcome should connect to your account.
    4. Click Finish
  5. Next, you will be asked to Map Welcome Fields to Hubspot Fields. If you are okay with the default field mapping configuration You can just click save.
  6. You can always update your Hubspot Authentication or field mappings by clicking Edit and selecting which settings you want to update from the dropdown.
  7. You can remove the connection to Hubspot and your field mapping configuration by clicking Reset. This will not delete any data. You can always connect to Hubspot and set a new field mapping configuration again.

 

Setup – Mapping Welcome Fields to Hubspot Contact Fields

Note: if you were using the previous version of our Hubspot integration which leverages the built-in Welcome Properties – Invited Welcome Events, Registered Welcome Events, and Attended Welcome Events – do not map data to these fields. 

 

  1. The default field mapping provided works for the standard Hubspot instance so if you don’t need any specific field mapping you can click “Save” to finish configuring the Hubspot integration. You can always come back to the field mapping configuration later.
  2. All Welcome fields that will map to Hubspot Contacts are visible on this page. The integration will send other Welcome data to Hubspot in the form of timeline activities on Contact objects. (See the “What data gets synced and when” section below). 
  3. Not all Field mappings are configurable by you. The destination of those mappings that you cannot update are in black boxes. The destination of those mappings that you can update are in grey boxes.
  4. When you click a mappable field to update you will be taken to a page to search for the field in Hubspot that you can map to. If you need to create a new field in Hubspot, first, create the field, then click “Select Mapping” in the Welcome Field Mapping Table.
  5. You must also select the Update Policy for a field. You can choose from: “Always Update”, “Update If Empty”, and “Don’t Update”.
  6. Custom Registration Fields:

    1. You can map up to 5 attendee custom field inputs on an event registration page.
    2. For example, if the custom field is "City", the integration will sync the attendee's answer: "New York"
    3. The custom fields are mapped in order
    4. Mapping custom fields is optional. 
    5. You should come back to the field mapping page to update the mapping configuration for different events which have different custom fields

Part 2: Marketing Events

  • When someone registers for a Welcome Event, this integration will find or create a Hubspot Marketing Event and subscribe the registering contact to that Marketing Event as “Registered” in real time
  • When someone attends the event, this integration will subscribe the contact to the Marketing Event as “Attended” in real time
  • Marketing Event subscriptions can be used to create lists in Hubspot 

Part 3: Contact Properties

When someone registers for your Welcome event, this integration will update their Hubspot Contact with the properties you have configured in the field mapper during integration set up. The update will occur in real time and will respect the Update Policy of the individual fields you have set in the field mapper.

 

How do I send registrants into Welcome from Hubspot

To send Contacts into Welcome such that they don’t need to register again, we recommend using our Quick Registration link functionality. 

 

Alternatively, you can create a Hubspot workflow to send registrants into Welcome, populate a Unique Link Property on each registrant’s Contact Object, and use that property your pre-event communications. To do so, do the following:

    1. Navigate to your Organization Integration settings.
    2. Click Edit Field Mapping
    3. Ensure that you have selected a mapping for a registrants’ Unique Links.
    4. Otherwise, create a Contact property in Hubspot for the Unique Link and select that property in the field mapper
    5. Navigate back to your integrations settings and Copy the webhook url listed
    6. In Hubspot create a contact based Workflow with your event’s Hubspot registration form as the trigger.
    7. Add a step to Send a Webhook
    8. Choose the POST Webhook option
    9. Paste the URL you copied from the Welcome Event Dashboard into the Webhook URL field
    10. Navigate to the Welcome event that you want to send registrants to and copy the Event ID of that event. (It’s the 6 characters in the event link after app.experiencewelcome.com/events/) 
    11. Replace the “ReplaceWithYourYourEventId” String in the Webhook URL with the actual Welcome event ID. In this case “wnuK0Z”
    12. Click Save and Publish your workflow
    13. Now when someone fills out your Hubspot registration form they will appear in the attendees list of your Welcome event
    14. Their Contact object will also contain their unique link to get into the Welcome event. This link will be stored in the Contact property you set for the Attendee’s Unique Link in the Field Mapping Configuration step above.
    15. You can use the Unique Link Property in your Hubspot emails as a personalization token.
