Integrations - Splash - Implementation

Splash / Welcome Integration Overview

Welcome offers an integration that connects a Splash Event to a Welcome Event, syncing registrant data between the platforms. 

The integration makes it easier for registrants who sign up on a Splash Registration page to navigate and check in to a Welcome virtual event, with the intent to maximize attendance at your Welcome event.

The following steps will allow a Splash administrator to send Event registrants into the Welcome platform. After a Splash guest is sent to Welcome, the guest’s information in Splash is updated with a unique event link for them to access the event seamlessly. This unique link can be used as a personalization token in event reminder emails.

The integration will create new Splash Guests if there are people who checked in to the Welcome Event but were not in Splash.

Lastly, the integration will check for updated attendee information on a daily basis. It will update all splash guests with their most up-to-date registration status at that time.

 

Before We Begin

* Enabling this integration requires the "Producer" role in Welcome

* Enabling this integration requires the "Admin" role in Splash. A Pro or Enterprise level Splash account is required to implement the full functionality of the integration, as described above. 

 

Steps 1-8 explain how to configure Splash so that the integration can populate a Splash Event with a Welcome Unique Link for each guest.

 

1. Contact your representative at Splash and request the following:

  • API Client ID
  • API Client Secret

2. Provide the above information to your Customer Success Manager or Solutions Engineer at Welcome, as well as login credentials for your Splash environment.

3. Within Splash, click on the event you want to integrate with Welcome and navigate to the "Settings" page.

 

4. In Splash, in the "Event Information" tab, scroll down to find the "Tags" field.

5. In the "Tags" field, please input the event "ID" for the event in Welcome you would like to associate the Splash event with. You can find the event "ID" in the Welcome platform by navigating to the Event Dashboard in Welcome. The event "ID" is a string of 6 characters that immediately follows "https://app.experiencewelcome.com/events/" in the event URL. This is how the Welcome platform will recognize which event in Splash it will need to send registrant data back to.

In the example screenshot below, the event "ID" is "nyuW1Z".

 

Within Splash, It is important to note that you must hit the ENTER key on your keyboard as well as click the "save" button on the top right-hand corner of the Splash page in order for the tag to be saved properly.

 

We now need to create a custom field in the event registration that aggregates the custom, unique URL for each attendee. This will ultimately go out in an email so that the attendee can seamlessly be navigated to the event occurring in Welcome with one click of the mouse. Steps 6-9 below describe one method to create this field, but there are other methods. Please engage with a Solutions Engineer or Customer Success Manager to further discuss.

6. Navigate back to the "Event" and click on the "Registration Form" tile on the center screen in Splash.

7. On the right-hand side of the screen, click the "Add Question" button at the bottom of the "Content" tab. Click the "Create New" tab and select the "Text (single line) option and click "Add to Form".

                              

 

8. Please type in "Unique Link For Welcome Event" in the Label field exactly. It is important to emphasize that we must use this exact, case sensitive syntax. You will notice that the "Field Name" field will auto-populate as well.

9.  Click the "Edit Logic" button on the bottom right-hand corner. Click "Add Condition" in the new window. The logic should be as follow: Unique Link for Welcome Event will be hidden unless <First Name> equals "Hide Always". Click Done.

 

 

 

Steps 10-13 are required to properly send registrant data from the Splash event to the Welcome event so that the guest does not need to re-enter any information when they check into Welcome.

10. Navigate back to the "Settings" page for the event. Click the "Integrations" tab. Click the "New Integration" button. A few tiles will pop-up, scroll down and select "HTTP".

 

11. In the Authentication tab, confirm that "Authentication Type" is set to "No Authentication". Then, Click Request Settings. In the Endpoint section, choose "POST" as the "Method". A Welcome Solutions Engineer and/or Customer Success Manager will provide a URL to fill in the "Postback URL" field at this point. Choose "Content-Type"  as "application/JSON".

 

 

12. In the "Triggers" section, please select "Attendee" as the trigger, and check the "RSVP Yes" box. This is what will allow Splash to send the updated statuses of each invitee to Welcome in real-time.

 

13. In order to properly deliver attendee data to the Welcome platform, we must map the attendee's contact information in Splash to the proper fields within Welcome. It is important to be aware that First Name, Last Name, and Email Address must be mapped at a minimum. It is also important that while "Splash Field" names can be different we must use the exact field names under the "Remote Field" column, as below:

Splash Field Remote Field
First Name first_name
Last Name last_name
Email email
Job Title company_role
Contact Company, [contact.company] company

 

Example Screenshot:

Once all the fields are properly mapped, please click the "Save" button.

 

14. Let's go back to the Event Dashboard in Welcome. Please click on the "Integrations" tab on the left-hand side and click "Trigger Integration". Once this is clicked, the integration is complete! Every day, the integration will automatically find all registrants in the Welcome Event and update the Guests of the associated Splash Event with their most up to date registration status and their unique link. the integration will create new Splash Guests for email addresses it cannot find as existing Guests of the Splash Event. We recommend manually triggering the integration around three hours before the event so that we can ensure the highest probability of all attendees receiving their unique link.
