3/25/22

Welcome back for another round of updates, fixes, and features!

Features

  • BETA:
    • Safari support is slowly being rolled out. Previously on desktop Safari, you had a limited experience where you can just watch the stream and users might be blocked from entering the app if the stream is not spun up yet. This feature makes the desktop Safari experience like the normal desktop experience on Chrome, Firefox, and Edge. The mobile experience using Safari is unchanged.
      • Some things to note:
        • The Jump In Screen now is "Click Anywhere to Jump In".
        • There is a dismissible banner directing users to use Chrome for the best experience
        • You can do everything on Safari (speak, produce, go to lounges/breakouts, etc.) except for screen share. We recommend that you still advise speakers and producers to use Chrome.
    • Quick registration allows for an easier joining experience for new attendees. Please contact support@experiencewelcome.com if you'd like this enabled for your organization.
    • Anonymous Q&A gives producers the ability to allow attendees to hide their names or to hide all names by default in Q&A. Please contact support@experiencewelcome.com if you'd like this enabled for your organization.
    • Domain deny list allows for producers to restrict registration from emails of a certain domain. Please note that this adds new behavior to sign out people who are denied entry into the event so that they can easily log in again with different credentials.

Enhancements

  • Allow for Scalable Vector Graphics (SVG) to be uploaded as backgrounds and overlays. You can use these files to give your event some extra pizazz with moving images. Please make sure that your SVG files follow our Dimension Guide for the best experience. 
  • Added in some recording monitoring improvements on our side.

Fixes

  • Producers will now see a timestamp of when a scheduled reminder email was sent or when it will be sent.
  • Added a fix for producers sometimes getting an error registering for events.
  • Fixed an issue where speakers who used a quick-reg link would automatically become attendees. Now when they use a quick-reg link they just land on the main event page to log in normally. Please note, there is no quick-reg “one-click” experience for speakers; we recommend speakers should use our standard one-click links for the best experience.
  • Added a fix for some cases where iPad users would not redirect to the mobile experience.
  • Added a fix for videos loaded into an event that would sometimes revert to a "processing" state.
  • Powered By Welcome logo
    • Made this logo the same color as the text in the footer.
    • Added the logo to partner events on the mobile experience.
